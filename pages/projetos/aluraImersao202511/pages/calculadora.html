Caluladora simples

A calculadora implementada é resposável por fazer a operação simples de apenas dois valores. Tentar adicionar mais de dois valores ou mais de uma operação não terá efeito algum e os erros devem estar tratados de algum modo a ser percebido pelo usuário.

Dentre as funcionalidades aceitas estão: adição, multiplicação, divisão, subtração, calcular resultado e limpar todo o campo (resetar). Dessa forma, pode-se notar que não tem a funcionalidade de apagar apenas um digito (normalmente o anterior) ou excrever tal operação com o próprio teclado (seja do PC ou Smartphone).

JavaScript e coleta de dados

Para resgar um bloco completo de informações do HTML, tal bloco tem que ter algumidentificador para ser diferenciado dos demais, tal como o id ou class.

O "id", por sua vez, pode ser resgatado por document.getElementById("seuIdAqui") onde "document" é tudo o que hpa dentro do arquivo html e "getElementById" é o que será procurado e resgatado do HTML (o id).

Seguindo essa mesma lógica, para pegar o identificador class de dentro do html seria o document.getElementByClass("suaClassAqui"), mas essa funcionalidade não existe.

Antes de falar sobre como pegar um ou mais blocos do HTML com uma mesma classe, é importante saber que classes podem ser identificadas com o ponto final seguido do nome da classe, como .suaClasseAqui e o id com um jogo da velha, como #seuIdAqui. Além disso, é importante citar que se dá um ID apenas para blocos únicos em todo o arquivo HTML, enquanto se pode atribuir uma mesma class para mais de um bloco. Em resumo, um ID para um único bloco e CLASS para um ou mais blocos.

Sabendo disso, para resgatar a primeira ocorrência de uma class (pode ter mais de uma), se usa document.querySelector(".suaClasseAqui"). Percebe-se que, diferentemente do id, é necessário informar o "." logo antes do nome da class. Com essa funcionalidade se pode pegar tanto class como id. Para pegar mais de uma class, use o comando document.querySelectorAll(".suaClassAqui")

Importante citar que essa funcionalidade também suporta o mapeamento de elementos como "#tabuleiro .pecas .peao", caso o arquivo HTML seja muito grande, facilitando a manutenção do código posteriormente, principalmente em casos de problemas.

Agora que sabe como foi feito o resgate de elementos do HTML para o JS, como os botões (na verdade são áreas que posteriormente foi adicionado evento de clique) e o campo de texto, falta saber como identificar qual botão foi acionado. E é bem simples, no HTML5. Para isso, veja o exemplo simples do Site Medium ou se preferir, na MDN Web Docs.
