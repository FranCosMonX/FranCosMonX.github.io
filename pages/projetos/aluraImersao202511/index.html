Projeto para a Imersão Alura de novembro 2025 com uso de IA (Germini)

Site criado com o objetivo de passar por todas as etapas propostas pelos instrutores da Alura Cursos. Essa aplicação será limitada apenas as tecnologias vistas nas aulas para manter o equilibrio entre os demais participantes (iniciantes), mantendo uma competição mais justa.

Tenho, por objetivo, ao finalizar esta imerssão, ter maior independência do uso de frameworks para consumo de APIs de terceiro, aumentando a possibilidade de manter futuros projetos mais leves e menos vulneráveis a erros, além de práticar um pouco a base HTML, CSS e JavaScript.

Será adotado o desafio de fazer todas as aplicações proposta na imerssao funcionar em várias telas (responsividade), uma vez que isso se tornou o novo padrão dos dias atuais. Outro é manter o código mais enxuto possível, evitando complexidade na manutenção deste.

Uma habilidade a ser treinada: uso de JavaScript para mudar elementos do HTML, evitando a criação de muitos arquivos HTML e códigos repetido.

Sobre o ambiente de desenvolvimento

Os códigos serão desenvolvidos na IDE Visual Studio Code (VSCode) usando a extenção Live Server para a atualização dinâmica do site ao salvar as alterações do código (não precisando utilizar o reload do navegador).

Sobre a localização do código

Todo o código poderá ser encontrado em um repositório do Github. Para quem necessita entender o mínimo dos comandos para armazenar os códigos no github, segue um mini tutorial:

Tutorial: instalando a ferramenta do Github chamada Git

Para conseguir utilizar o github sem problemas é necessário instalar o Git e configurá-lo. Para instalar, utilize o link a seguir: git-scm.com/install/. Ao terminar a instalação, execute o Git Bash e utilize os seguintes comandos contendo as mesmas informações contidas em sua conta do GitHub:
git config --global user.name "Fulano de Tal"
git config --global user.email fulanodetal@exemplo.br Mais informações podem ser encontradas no link a seguir: https://git-scm.com/book/pt-br/v2/Come%C3%A7ando-Configura%C3%A7%C3%A3o-Inicial-do-Git.

Importante informar que será solicitado que se autetique ao Github na primeira vez que utilizar um primeiro comando. O passo a passo é guiado pelas plataformas, é só ler e seguir cada passo informado.

Tutorial: criando um repositório do Github

Para criar um repositório do Github é necessário ter uma conta na própria plataforma. Para isto, clique no link a seguir para ser redirecionado ao site para o login ou criação da conta: https://github.com/login

Feito isso e estando na página inicial, clicar na imagem do perfil no canto direito superior >> repositories/repositórios e logo a seguir clicar em um botão verde "new" / "novo".

Nesse tutorial, é importante citar que se você, que está lendo, tiver um código já pronto, deve desmarcar a opção de adicionar um arquivo README. Caso você não tenha um código, pode manter a opção marcada caso queira somente clonar o repositório localmente na máquina.

Ao apertar o botão "Criar repositório" / "Create repository", irá aperecer dois conjuntos de códigos para quem optou desmarcar a opção de adicionar o arquivo README usar. Um dos conjuntos é usado para criar um repositório localmente na máquina e enviar um commit ao criar um arquivo README básico, e o outro é para quem já possui um projeto e deseja vinculá-lo a um repositório.

Para qualquer um desses, é necessário estar dentro de um diretório de arquivos que esteja ou estará armazenado os arquivos do projeto para em seguida, clicar com o botão direito do mouse em qualquer área vazia desse diretório e selecionar (windows 10) Open Git Bash Here (Abrir git bash aqui) / (Windows 11) mostrar mais opções >> Open Git Bash Here (Abrir git bash aqui).

Feito isso, a depender do seu objetivo, coloque comando por comando no terminal do Git Bash que deverá ter aparecido e pressione 'Enter'. Ao finalizar, volte para o repositório do Github (no navegador) e atualize a página. Todo o código deverá estar contido nele. Vale relembrar que, se for o primeiro comando a ser executado após a instalação do Git, irá aparecer um passo a passo para que seja feito a autenticação com a conta do Github.

Tutorial: Enviando alterações para o repositório do Github

Caso seu projeto já esteja ligado a um repositório remoto do Github, o envio das alterações é bastante simples.

'git add .' para empacotar todas as alterações salvas.
'git commit -m "escreva o porque está commitando"' para preparar todo esse pacote de alterações para ser enviado posteriormente. Note que tem um "-m" seguido de uma string para impedir que seja aberto algum outro editor de texto no terminal, aumentando a complexidade do commit para iniciantes a git.
'git push' para enviar todos os pacotes de alterações salvos localmente para o repositório remoto (armazenado no Github).
